## 什么是 PMP？

**PMP (Project Management Professional)** 是由美国项目管理协会 (PMI) 发起的全球最具权威性的项目管理专业人士资格认证。它不仅是一张证书，更是项目管理能力的国际认可。

### PMP 的价值

| 价值点 | 说明 |
|--------|------|
| 🌍 **国际认可** | 全球 200+ 国家认可，项目管理领域黄金标准 |
| 💰 **薪资提升** | PMP 持证者平均薪资比非持证者高 20%+ |
| 📈 **职业发展** | 提升项目管理能力，拓展职业发展空间 |
| 🤝 **人脉资源** | 加入全球 PMP 社区，拓展专业人脉 |

## PMP 核心概念体系

PMP 基于项目管理知识体系指南 (PMBOK)，包含五大过程组和十大知识领域：

### PMP 概念全景图

<img src="/images/pmp-concept.png" alt="PMP 核心概念图" style="border-radius: 8px;" />

### 五大过程组

| 过程组 | 核心作用 | 关键词 |
|--------|----------|--------|
| **启动过程组** | 定义新项目或新阶段 | 授权、章程、识别干系人 |
| **规划过程组** | 明确项目范围和行动计划 | 计划、策略、基准 |
| **执行过程组** | 完成项目工作 | 指导、管理、质量保证 |
| **监控过程组** | 跟踪进展并采取纠正措施 | 监控、控制、变更 |
| **收尾过程组** | 正式结束项目或阶段 | 验收、总结、释放 |

### 十大知识领域

| 知识领域 | 简称 | 核心内容 |
|----------|------|----------|
| **项目整合管理** | Integration | 统筹协调各领域，确保项目整体成功 |
| **项目范围管理** | Scope | 明确做什么、不做什么，控制范围蔓延 |
| **项目进度管理** | Schedule | 制定并控制项目进度计划 |
| **项目成本管理** | Cost | 估算、预算和控制成本 |
| **项目质量管理** | Quality | 确保交付成果满足质量要求 |
| **项目资源管理** | Resource | 获取、开发和管理项目团队 |
| **项目沟通管理** | Communication | 确保信息及时准确地生成和传递 |
| **项目风险管理** | Risk | 识别、分析和应对风险 |
| **项目采购管理** | Procurement | 从外部获取产品、服务或成果 |
| **项目干系人管理** | Stakeholder | 识别和管理干系人期望 |

### 关键概念解析

#### 1. 三重约束 (铁三角)

项目管理核心约束，相互影响、相互制约：

```
        ┌─────────┐
        │  范围   │
        │ Scope   │
       /└────┬────┘
      /      │
     /       │
    /        │
  时间 ──────┴────── 成本
  Time            Cost
```

#### 2. 项目生命周期

| 类型 | 特点 | 适用场景 |
|------|------|----------|
| **预测型** | 明确范围和计划，需求稳定 | 建筑、制造业 |
| **迭代型** | 重复循环，逐步完善 | 研发、设计类 |
| **增量型** | 分批交付，快速反馈 | 软件产品 |
| **适应型** (敏捷) | 拥抱变化，快速响应 | 互联网产品 |
| **混合型** | 结合预测和适应 | 复杂项目 |

#### 3. 项目经理角色

PMP 强调项目经理是 **整合者** 和 **领导者**：

- **整合者**：协调各方资源，统一项目目标
- **领导者**：激励团队，营造积极氛围
- **沟通者**：确保信息流畅传递
- **决策者**：及时做出关键决策
- **促进者**：帮助团队解决问题

### 学习建议

#### 1. 理解大于记忆

PMP 考试侧重 **应用和理解**，而非死记硬背：

- ✅ 理解流程和输入输出
- ✅ 掌握工具和技术的应用场景
- ✅ 熟悉各过程的关联关系
- ❌ 不要只背术语和公式

#### 2. 场景化思考

考试题目多为场景题，需要代入项目经理角色：

> **典型考题模式**：
> "你是某项目的项目经理，在执行阶段遇到...情况，你应该首先怎么做？"

**解题思路**：
1. 明确当前所处的项目阶段
2. 识别问题类型（范围、进度、风险...）
3. 回顾对应的知识领域处理流程
4. 选择最符合 PMI 思维的选项

#### 3. PMI 主义思维

考试以 PMI 的价值观为准，需要培养 **PMI 思维**：

- **以流程为导向**：遵循标准流程，不凭直觉行事
- **以干系人为中心**：关注干系人期望和满意度
- **积极主动**：预防优于补救，主动沟通
- **数据驱动**：用数据和事实说话
- **团队协作**：授权团队，集体决策

### 考试技巧

#### 时间管理

- 180 道题，230 分钟
- 平均每题约 1.2 分钟
- 建议每 60 道题检查一次时间

#### 答题策略

1. **先易后难**：标记不确定的题目，做完再回看
2. **排除法**：先排除明显错误选项
3. **相信第一直觉**：不要轻易改答案
4. **注意关键词**：首先、最后、最佳、除外

## 考试形式 (2024 版)

### 新考试内容

PMP 考试已于 2024 年更新，采用 **ECO (Exam Content Outline)** 标准：

| 领域 | 占比 | 说明 |
|------|------|------|
| **人员** | 42% | 团队管理、领导力、冲突解决 |
| **过程** | 50% | 项目管理流程、整合、范围/进度/成本 |
| **商业环境** | 8% | 合规、组织变革、项目交付价值 |

### 考试特点

- **混合题型**：单选题 + 多选题（会标注）
- **情景为主**：约 80% 为场景题
- **敏捷比重增加**：预测型约 50%，敏捷/混合型约 50%
- **180 道** + 25 道预选题（不计分）
